40+ connectors · hybrid search + rerank · bring your own keys · your Postgres

A real memory backend

Not a wrapper. A store you own.

Everything you drop in is chunked, embedded, and indexed with a vector and a full-text signal side by side. Retrieval fuses both, so you get meaning and exact keywords.

Hybrid search, ranked well

pgvector similarity and Postgres full-text, fused with reciprocal rank fusion, then optionally reordered by an LLM.

Answers with receipts

RAG chat streams answers grounded in your knowledge, each citation pointing back at the exact source passage.

Reads more than text

OCRs images, transcribes audio, extracts PDFs, pulls video transcripts.

Stays organized

Auto-summaries, tags, and facts on ingest. Dedup, version history, signed webhooks.

Agent-native

A built-in MCP server. Claude, Cursor, and any agent read and write the brain as a tool.

Living playbooks

Ask for a topic and CompanyBrain drafts a cited page from your memories — decisions, process, people, open questions. Regenerate it and it reflects what the team knows now.

Temporal memory

Knowledge changes. When a new memory updates a fact already on record, the old one is marked superseded — so recall returns the current truth while the history stays.

Ingest, embed, retrieve, cite

One pipeline from any source to a cited answer. Bring your own Postgres; nothing phones home.

Who it's for

Memory for every kind of work.

The answer you need already exists somewhere in the work your team did. CompanyBrain makes it findable.

Engineering

The reason behind a strange line of code is usually a PR review or a thread from six months ago. Index the repo, the reviews, and the chat, then ask why.

Sales

Objections, promised follow-ups, and "let me check with the team" moments scatter across calls and threads. Point CompanyBrain at them and pull the thread when a deal needs it.

Support

The fix a teammate found last quarter should not be rediscovered from scratch. Search resolved tickets and docs, not your memory.

Ops & leadership

Why did we pick this vendor? What did we agree in planning? The answer becomes a search with citations, not another meeting.

Index everything

Forty-plus connectors, one store.

Configure and sync any source from the dashboard. It all lands in the same place, so search and chat work across every source at once.

One brain, one team

Built for you. Ready for your whole team.

Start as one person with no login wall. Turn on shared workspaces when the team shows up. The memory, the connectors, and the agents all come along.

For you

Single-user mode is the default: no sign-in, one workspace, everything you index is yours. Capture notes, connect your own tools, ask questions, and draft playbooks from your own work. Delete anything, any time.

For your team

Flip on multi-workspace auth and the same brain scales: shared spaces, API keys, roles, and the decision behind every doc. Everyone searches one source of truth, and the why is tagged to the thread it came from.

Record in one agent, recall in another

The built-in Model Context Protocol server means Claude, Cursor, and your own tools read and write the same memory. Context stops living in one chat window and starts following the work.

claude → record → companybrain → recall → cursor any MCP client writes once, every client reads it back
