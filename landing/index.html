Two ways to find. One ranked answer.

Every passage you store carries a vector for meaning and a full-text signal for exact words. A query runs both lanes at once, then reciprocal rank fusion merges them into a single ordering. An optional reranker reads the top candidates and settles the final order.

  • Meaning and keywords together. Semantic recall finds the idea; full-text catches the exact identifier, code symbol, or name.
  • Fusion, not a toggle. Weighted RRF blends both rankings, so a result strong on either signal surfaces.
  • Reranked for precision. HyDE query expansion and an LLM reranker push the right passage to the top.

A real memory backend, not a wrapper.

Everything you drop in is chunked, embedded, indexed, and kept current. These are the parts that make recall trustworthy.

Temporal memory

Knowledge changes. When a new memory updates a fact already on record, the old one is marked superseded, so recall returns today's truth while the history stays intact.
