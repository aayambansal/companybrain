The open-source memory layer for your company

Point it at your knowledge — Obsidian, Google Docs, Slack, Notion, GitHub, PDFs, the web — and get one private, searchable brain that answers with citations. Your agents read it over MCP. Your data never leaves your Postgres.

$ npx companybrain
MIT licensed · no sign-in · bring your own keys · 32 connectors

What it does

A real memory backend, not a wrapper

Everything you drop in is chunked, embedded, and indexed with a vector and a full-text signal side by side. Retrieval fuses both, so you get meaning and exact keywords.

◈

Hybrid search

pgvector similarity plus Postgres full-text, fused with reciprocal rank fusion, and optional LLM reranking.

❝

Answers with receipts

RAG chat streams answers grounded in your knowledge, with citations that point back at the source passage.

⌘

Agent-native

A built-in MCP server lets Claude, Cursor, or any agent read and write the brain as a tool. Keyless locally.

⚿

Bring your own keys

Runs zero-config on a local model. Paste an OpenAI, Anthropic, Google, or Ollama key to upgrade, live.

◱

Yours to run

One command with Docker. Bring your own Postgres. No hosted control plane, nothing phones home.

↻

Stays fresh

Connectors sync on a schedule. Change the embedding model and re-index everything from the dashboard.

How it works

Ingest, embed, retrieve, cite

                                                    │
   obsidian · gdocs · slack · notion · github       ▼
   web · rss · pdf · linear · confluence · jira   postgres + pgvector  (your database)

Index everything

Thirty-two connectors, and counting

Configure and sync any source from the dashboard. Everything lands in the same store, so search and chat work across all of it without caring where it came from. New memories can be auto-summarized, tagged, and deduped, and fire a signed webhook.
